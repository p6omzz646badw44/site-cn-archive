# site-cn-archive

本项目用于归档和发布多个独立的 HTML 页面。仓库本身不针对任何特定域名或网站，仅作为静态页面集合的存储与分发空间。

## 项目简介

site-cn-archive 是一个通用型 HTML 页面归档仓库。所收录的页面均以独立文件形式存在，彼此之间无程序依赖或站点关联。

## 目录说明

```
site-cn-archive/
├── pages/          # 存放归档的 HTML 页面文件
├── assets/         # 页面引用的资源文件（如图片、样式、脚本等）
└── README.md       # 本文件
```

- **pages/**：归档的 HTML 页面。每个文件代表一个独立页面，命名遵循内容描述或时间标识。
- **assets/**：与页面配套使用的辅助资源，按需存放，保持目录结构清晰。

## 页面归档说明

- 所有页面均为静态 HTML，可直接通过浏览器或 GitHub Pages 访问。
- 归档内容不涉及动态交互、后端服务或外部数据接口。
- 页面内容可能随时间更新，历史版本可通过 Git 提交记录追溯。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面会通过常规 Git 提交完成。
- 欢迎提交 Issue 或 Pull Request，但请勿包含具体域名、网址或营销推广信息。
- 页面内容可能来自多个来源，归档仅作技术性保存，不代表对内容本身的认可或背书。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-cn-archive.git
   ```
2. 直接双击 `pages/` 目录下的 HTML 文件即可在浏览器中打开查看。
3. 若启用 GitHub Pages，可设置发布源为 `main` 分支的根目录或 `/docs` 目录，随后通过对应 URL 访问。
